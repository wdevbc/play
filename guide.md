# guide.md
## 안티그래비티(Antigravity) IDE 투입용 공통 설계 가이드라인 (GitHub Pages 정적 사이트 버전)

본 파일은 안티그래비티 IDE에 그대로 복사·붙여넣기 하여 여러 종류의 자산 및 투자 티어 테스트 서비스를 GitHub Pages 환경에서 구동 가능한 정적 HTML/CSS/JS 단독 앱으로 자동 빌드하기 위한 가이드라인입니다.

---

### 🚀 공통 인프라 및 아키텍처 명세

[프로젝트 기본 인프라 설정]
1. 본 프로젝트는 백엔드 서버(Node.js 등) 및 Docker 없이 오직 **정적 HTML, CSS, JavaScript**만을 사용하여 구현됩니다.
2. 배포 환경은 **GitHub Pages**를 사용합니다. 별도의 무중단 배포 서버 없이 main 브랜치 푸시만으로 정적 페이지 배포가 완료되도록 구성합니다.
3. 구글 애널리틱스(GA4) 등 분석 툴 연동이 용이하도록 표준 HTML5 구조를 엄격히 준수합니다.

[UI/UX 및 레이아웃 제약조건]
1. 카카오톡 오픈채팅방 공유 및 카톡 인앱 브라우저 최적화를 위해, 프론트엔드 최외곽 래퍼(wrapper)는 최대 가로폭 800px 이하로 고정하고 중앙 정렬해줘. (max-width: 800px; margin: 0 auto;)
2. 모바일 퍼스트 뷰포트를 지원하며, 무겁고 몰입감 있는 다크 모드 톤앤매너를 유지해줘. 포인트 컬러로는 딥블루와 골드 네온 효과를 사용해줘. (CSS 구조는 BEM 방법론을 지향)
3. 결과 페이지(Result)의 하단 또는 중앙에는 직관적인 엠블럼 이미지 영역과 함께, 향후 카카오톡 공유 API(Kakao.Link)와 연동할 수 있는 바인딩용 "카카오톡으로 티어 인증하기" 버튼을 UI 레이아웃에 필수로 포함해줘.

[프로젝트 폴더 구조 구성안]
여러 개의 테스트 서비스가 독립적으로 추가될 수 있도록 폴더 구조를 아래와 같이 구성합니다:
- `/index.html` (루트 메인 홈 페이지 - 향후 여러 테스트 리스트를 모아두는 허브 페이지)
- `/[서비스명]/index.html` (각 서비스별 질문 진행 프론트 페이지, Progress Bar 포함)
- `/[서비스명]/result.html` (각 서비스별 티어 결과 출력 및 카카오톡 공유 버튼 포함 페이지)
- `/[서비스명]/css/style.css` (각 서비스별 전용 스타일시트)
- `/[서비스명]/js/tierCalculator.js` (각 서비스별 질문 데이터 및 판별 알고리즘 JS 모듈)
- `/[서비스명]/guide.md` (각 서비스별 개별 질문 데이터 및 전용 알고리즘 가이드)
